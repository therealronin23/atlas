# absorb-without-cloning

## Why

Atlas mejora absorbiendo trabajo externo útil, no copiando identidades de otros
productos. Cursor, Codex, Claude Code, MemGPT/MemPalace, MCP servers, skills,
prompts, APIs y repos externos son material de aprendizaje: se diseccionan, se
extrae lo que sirve, se combina con piezas de otros sitios cuando conviene y se
envuelve con las invariantes de Atlas.

## How to apply

- No hacer forks agresivos ni clones de producto salvo que el usuario lo pida
  explícitamente.
- Evaluar cada sistema externo por piezas: capacidad, interfaz, patrón operativo,
  verificador, UX, memoria, routing, permisos, reversibilidad y coste.
- Absorber solo lo que encaja con Atlas: Merkle, Decider, sandbox, capabilities,
  memoria, verificación y autonomía gobernada.
- Mantener separadas las superficies: el tronco MCP es una interfaz del ecosistema,
  no el contenedor de todo el ecosistema.
- Cuando una parte externa sea mejor que la nuestra, adoptarla o envolverla con
  mínima cola propia; cuando solo sea inspiración, registrar el patrón y no crear
  código.
