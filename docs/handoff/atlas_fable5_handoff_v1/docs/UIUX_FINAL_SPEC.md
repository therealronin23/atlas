# Atlas OS — UI/UX Final-Compatible Spec

## Dos caras obligatorias

### 1. Cognitive Surface

Donde Atlas piensa, ejecuta, recuerda y muestra su mundo vivo.

Componentes:

- Living Knowledge Graph.
- Universal Bar.
- Execution Pipeline.
- Timeline / Audit Timeline.
- Memory Vault.
- Artifacts.
- Territory Switcher.
- Reality Panel.

### 2. Control Plane

Donde el usuario gobierna qué puede tocar Atlas.

Componentes:

- Control Center.
- Integration Fabric.
- Accounts & Identity.
- Permissions Matrix.
- Personalization.
- Notification Router.
- Automation Rules.
- Model & Provider Router.
- Security Center.
- Backup / Export / Portability.
- Developer Console.

## Home

La home no es un chat. Es el Living Knowledge Graph.

El chat es una modalidad:

- Universal Bar para comandos breves.
- Conversation panel bajo demanda.
- Conversation nodes dentro de memoria.

## Integration Fabric UX

Cada integración debe tener:

- Estado.
- Tipo: source, capability, channel, memory feed, action surface.
- Permisos.
- Risk zone.
- Última sincronización.
- Memoria generada.
- Acciones permitidas.
- Acciones bloqueadas.
- Logs.
- Automatizaciones asociadas.
- Desconexión/revocación.

## Personalización

Debe incluir:

- Apariencia.
- Densidad de interfaz.
- Nivel de animación.
- Modo simple/pro/architect.
- Nivel de autonomía.
- Cuándo pedir confirmación.
- Qué recordar.
- Qué no recordar.
- Notificaciones.
- Canales.
- Modelos preferidos.
- Nivel de riesgo permitido.

## Criterio UX

Todo lo conectado debe convertirse en evento, memoria, nodo, proceso, artifact o capability. Si una integración queda como simple pestaña externa, está mal diseñada.
