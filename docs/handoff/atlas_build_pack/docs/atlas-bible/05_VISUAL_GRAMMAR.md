# 05 — Visual Grammar

## Paleta principal

```text
Background       #0A0A0A
Panel            rgba(255,255,255,0.05)
Border           rgba(255,255,255,0.12)
Cyan / Thought   #00F5FF
Indigo / System  #6366F1
Green / Valid    #10B981
Amber / Warning  #F59E0B
Red / Error      #EF4444
Text             #E5E5E5
Muted            #A3A3A3
```

## Significado

```text
Cyan     = pensamiento, actividad cognitiva, búsqueda, reflexión
Indigo   = estructura, sistema, kernel, soberanía
Green    = validado, completado, Merkle verified
Amber    = espera, aprobación humana, riesgo medio
Red      = error, bloqueo, contradicción, riesgo alto
White    = usuario, entrada humana, foco actual
Gray     = inactivo, archivado, baja relevancia
```

## Reglas de UI

1. Nunca más de tres focos visuales fuertes activos al mismo tiempo.
2. La home siempre debe mostrar estado vivo, no una lista muerta.
3. Los paneles laterales son auxiliares. El centro pertenece al estado cognitivo.
4. Cualquier badge debe significar algo verificable.
5. Los colores no decoran; codifican semántica.
6. Las animaciones no decoran; representan cambios de estado.
7. La interfaz debe poder degradar a modo sobrio si hay saturación.
