# 00 — Manifiesto de Atlas OS

## Definición

Atlas OS es un sistema operativo cognitivo soberano.

No es un chatbot.  
No es un IDE.  
No es un dashboard.  
No es un wrapper de herramientas de IA.  
No es un clon de n8n, Cursor, Claude Code, Codex, Odysseus ni Obsidian.

Atlas es el entorno donde el conocimiento, la memoria, las herramientas, los procesos, los artefactos, la auditoría y el usuario conviven como un sistema vivo.

## Tesis

El usuario no debe perder su conocimiento entre conversaciones, plataformas, modelos, proyectos y herramientas. Atlas convierte ese historial disperso en una memoria operativa, auditable y accionable.

## Diferenciador

Atlas no compite intentando ser "otro Claude" o "otro Cursor".

Atlas compite como el lugar donde todo lo que el usuario hizo con Claude, ChatGPT, Cursor, Copilot, Aider, Codex, repositorios, documentos y herramientas se convierte en conocimiento propio, persistente, navegable y reutilizable.

## Forma mental

```text
Intent
↓
Context
↓
Memory
↓
Plan
↓
Execution
↓
Observation
↓
Artifact
↓
Learning
↓
Audit
```

Este flujo no debe quedar oculto. Debe ser visible, controlable y auditable.

## Principios irrenunciables

1. El centro visual de Atlas es el Living Knowledge Graph.
2. La entrada principal es la Universal Bar.
3. El chat es una modalidad, no el producto.
4. El Execution Pipeline muestra cómo Atlas trabaja.
5. La Timeline muestra qué ocurrió y cuándo.
6. Memory Vault muestra qué aprendió Atlas.
7. Governance/Gates decide qué puede hacer Atlas sin aprobación.
8. Cualquier herramienta externa es un adapter, no el núcleo.
9. LangGraph, CrewAI, React Flow, Tauri o Slint no definen Atlas.
10. La UI debe poder funcionar con eventos simulados antes del backend completo.

## Resultado buscado

Una primera versión que se acerque lo máximo posible a la visión final: shell real, sistema de eventos real, grafo navegable real, pipeline reactivo real, bridge inicial con Atlas Core y arquitectura ampliable.
