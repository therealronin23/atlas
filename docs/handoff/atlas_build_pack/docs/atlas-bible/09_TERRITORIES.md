# 09 — Territorios

## Command Center

Home de Atlas. Muestra Living Knowledge Graph, estado del sistema, timeline y entrada universal.

## Coding Territory

Para tareas de código.

Debe incluir:

```text
- Proyecto/repo
- Editor Monaco
- Diff viewer
- Terminal sandbox
- Execution Pipeline
- Blast radius
- Tests
- Artifacts
- Memory context
```

## Research Territory

Para investigación profunda.

Debe incluir:

```text
- Árbol de preguntas
- Fuentes internas
- Fuentes externas
- Contradicciones
- Hipótesis
- Evidencia
- Síntesis
- Memoria nueva
```

## Memory Territory

Para ver y editar conocimiento.

Debe incluir:

```text
- Conversation imports
- Patterns
- Decisions
- Mistakes
- Preferences
- Project knowledge
- Provenance paths
```

## Visual Orchestrator

Territorio avanzado, no home.

Debe incluir:

```text
- Canvas tipo n8n, propio
- Node palette
- Inspector
- Graph compiler
- Templates
- Execute/debug
- Export/import graph
```

## Audit Territory

Para confianza y trazabilidad.

Debe incluir:

```text
- Merkle log
- Timeline
- Event replay
- Snapshots
- Approvals
- Rollback
```

## Connected Accounts

Para importar historial externo.

Debe incluir:

```text
- ChatGPT exports
- Claude exports
- Cursor exports
- Copilot history if available
- Manual import
- Account connector later
```

## Bond Territory

Conversación más humana o reflexiva, con acceso controlado a memoria y explicación bajo demanda.
