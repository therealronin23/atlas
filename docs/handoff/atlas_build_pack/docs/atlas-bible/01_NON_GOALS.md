# 01 — Non Goals

Estos son límites explícitos. Sirven para evitar que Atlas derive hacia otro producto genérico.

## No construir

- No construir un chatbot con sidebar.
- No construir un dashboard bonito de métricas.
- No construir un clon de Cursor.
- No construir un clon de n8n.
- No construir un clon de Obsidian.
- No construir un wrapper de Claude Code, Codex, Aider o Cursor.
- No construir una UI fija que dependa de APIs inmaduras.
- No construir un frontend que sea solo decoración.
- No poner el Visual Orchestrator como Home principal.
- No usar LangGraph como definición del kernel de Atlas.
- No hacer que React contenga lógica de dominio de Atlas.
- No crear 20 pantallas antes de validar el Event Canon.
- No generar más mockups sin contratos de datos.
- No ocultar el proceso detrás de "pensando...".
- No convertir la Biblia en una excusa para no construir.

## Reglas de prioridad

1. Si hay conflicto entre belleza y trazabilidad, gana trazabilidad.
2. Si hay conflicto entre framework y Event Canon, gana Event Canon.
3. Si hay conflicto entre chat y sistema visible, gana sistema visible.
4. Si hay conflicto entre velocidad y seguridad en acciones con efectos externos, gana seguridad.
5. Si hay conflicto entre "clonar algo existente" y crear identidad Atlas, gana identidad Atlas.

## Frase de control

Si una pantalla podría pertenecer a Cursor, n8n, Claude o Obsidian cambiando solo el logo, no es suficientemente Atlas.
