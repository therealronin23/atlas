# Acceptance Criteria

A Phase 15 result is acceptable only if:

1. Current web UI is documented as validation harness.
2. Product constitution exists in repo.
3. Research 01-15 docs exist or are mapped.
4. Schemas validate as JSON.
5. Fixtures exist for gestoría, external thought, workflows, visual states, security, integrations, connection recipes, connector packs and business core.
6. Integration Fabric contracts exist.
7. Connection Store/Concierge contracts exist.
8. Atlas Native Business Core contracts exist.
9. Adaptive Question Engine contracts exist.
10. UI Quality Gate exists and can reject weak UI.
11. Security tests include prompt injection, tool poisoning, memory poisoning, cloud-sensitive denial and high-risk remote command denial.
12. No real external dangerous action is implemented.
13. Continuation files are written.
14. At least 10 improvement proposals/gaps are produced.
