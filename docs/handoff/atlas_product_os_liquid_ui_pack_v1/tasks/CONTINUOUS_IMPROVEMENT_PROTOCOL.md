# Continuous Improvement Protocol

At the end of every Fable phase:

1. List what was built.
2. List what was not built.
3. List 10+ gaps or weaknesses.
4. Classify each gap: product, UX, backend, security, integration, data, legal, testing, performance, maintainability.
5. Fix safe gaps immediately.
6. Defer unsafe/large gaps with concrete next step.
7. Write `IMPROVEMENT_PROPOSALS.md`.
8. Update `KNOWN_RISKS.md`.
9. Update `NEXT_AI_INSTRUCTIONS.md`.

Improvement must strengthen Atlas. It must not simplify Atlas into a generic assistant or dashboard.
