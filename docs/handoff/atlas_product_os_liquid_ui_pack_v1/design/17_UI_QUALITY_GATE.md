# UI Quality Gate

Reject UI if:

- it looks like SaaS dashboard/card soup;
- it looks like cheap Jarvis;
- it looks like React template;
- nodes have no meaning;
- movement is decorative;
- there is no objective, sector, state, risk, evidence, inspector, timeline or Gate;
- command palette bypasses permissions;
- mobile is desktop compressed;
- reduced-motion is missing.

Each screen must answer:

1. Where am I?
2. What objective is active?
3. What is Atlas doing?
4. What data is used?
5. What risk exists?
6. What action can happen next?
7. Can I simulate/pause/cancel/reject?
8. Can I see evidence and audit?
